コントリビューター

チームと貢献について

Scheduly はコミュニティ主導のモックプロジェクトです。コントリビューターは、計画立案・UI 改善・ドキュメント整備・バグ修正など多岐にわたって協力しています。プロジェクトの進捗は GitHub のリポジトリで公開されているため、誰でも履歴を追いながら参加できます。

コントリビュートの始め方

  1. リポジトリをフォークし、ローカル環境にクローンします。
  2. docs/CONTRIBUTING.md に目を通し、スタイルガイドや開発フローを確認します。
  3. Issue から着手可能なタスクを探す、もしくは新しい提案を行います。
  4. 変更を加えたら Pull Request を作成し、背景や意図を説明してください。

GitHub Issues / Pull Requests の公開チャネルでは、機密情報を含めないようご注意ください。

コントリビューター一覧

最新のコントリビューターはリポジトリ内の docs/CONTRIBUTORS.md に掲載しています。コミット履歴からも貢献状況を確認できます。

連絡とサポート

プロジェクトに関する質問や提案は GitHub Issues を活用してください。Apache License 2.0 のもと、対応はベストエフォートで行われるため、必ずしも応答や修正を保証するものではありません。
